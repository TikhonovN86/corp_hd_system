# СЦЕНАРИИ УВЕДОМЛЕНИЙ ДЛЯ ИНИЦИАТОРА:

СЦЕНАРИЙ №1 - Получен комментарий от исполнителя или диспетчера.

Условие (Момент вызова):
        
    В методе addComment() сразу после сохранения комментария в БД.

Сценарий:

    - Проверяем: комментарий.приватный == false

    - Проверяем: автор.роль == ИСПОЛНИТЕЛЬ ИЛИ автор.роль == ДИСПЕТЧЕР

    - Если оба условия true -> создать уведомление для ticket.initiatorId

Альтернативный сценарий:

    Если комментарий приватный — инициатор не получает ничего (ни текста, ни факта существования комментария).

СЦЕНАРИЙ № 2 - Обращение решено

Условие (Момент вызова):

    В методе resolveTicket() при нажатии кнопки "Выполнить".

Сценарий:

    - Исполнитель или Диспетчер нажимает "Выполнить".

    - Создается уведомление для инициатора:

        "Обращение [id обращения] Решено: [Текст комментария]"
