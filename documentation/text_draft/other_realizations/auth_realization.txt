------------------------------------------------------------
# ЛОГИКА

- session-based auth через Spring Security.

- Что будет добавлено:
1. форма входа
2. таблица пользователей
3. хранение пароля в хэше
4. выдача сессии после логина
5. проверка роли и принадлежности к рабочей группе на backend

- Как все примерно работает:

1. Пользователь вводит логин/пароль;
2. backend проверяет их;
3. Если всё хорошо — создаётся HTTP-сессия;
4. Браузер хранит cookie с id сессии;
5. Каждый следующий запрос уже идёт от имени этого пользователя.

- Уточнения:

1. У пользователя может быть только одна роль.
2. Пользователь может состоять в одной или нескольких рабочих группах.
3. Авторизация реализуется через Spring Security session-based auth.
4. Доступ к данным проверяется на backend в сервисном слое.
