# РОЛИ

- Инициатор - Пользователь, который создает обращения. Видит историю своих обращений, может просматривать карточку обращения,
в которых он является инициатором. Обращения от других инициаторов не доступны для просмотра и редактирования.

- Диспетчер - Проверяет поступающие обращения на корректность заполнения (требуется минимальная экспертиза), 
может передать обращение на ответственную РГ или закрыть обращение.

- Испольнитель - Проверяет поступающее обращение, может вернуть обращение диспетчерам,
может добавлять комментарии, может установить статус "Решено".

# ОБЩАЯ СТРУКТУРА УСЛУГ

1. Раздел.

- Обозначает семейство вопросов, объединенных в один раздел.
Например: "Корпоративномый Help Desk".

2. Направление обслуживания (далее НО).

- У кождого раздела есть перечень своих направлений обслуживания.
Пример направления обслуживания для раздела "Корпоративномый Help Desk": "Создать новый раздел".

3. Тип обращения.

- Подробнее в # ТИП ОБРАЩЕНИЙ.

4. Статус обращения.

- Подробнее в # СТАТУСЫ.

5. Рабочая группа (Далее РГ)

- Подробнее в # РАБОЧИЕ ГРУППЫ

# СТАТУСЫ

- Статус не отражает владельца процесса, за владельца отвечает поле "Рабочая группа".

- NEW - Новое  - пользователь создал обращение, оно не принято в рабоу. Изначально попадает на группу диспетчеров.
- IN_PROGRESS - В работе.
- UNASSIGNED - На распределении (Исполнитель вернул на диспетчеров).
- RESOLVED - Решено. 

# МАРШРУТЫ СТАТУСОВ:

Новое

Диспетчер:

Новое - В работе

# ВАЖНЫЕ МОМЕНТЫ:

- Чтобы перевести обращение в "Решено": 
1. Всегда сначала нужно перевести обращение в статус "IN_PROGRESS".
2. Если карточке обращения отсутствуют не приватные комментарии от Исполнителя или Диспетчера,
обращение не закрывается, уведомление не отправляется, появляется модальное окно "Для решения обращения добавьте комментарий".
3. Должно быть заполнено поле "Ответственный сотрудник" - можно указать исполнителя из ответственной РГ, 
в т.ч. диспетчера, если РГ - help.desk.dispetcher. 

- Диспетчер из статуса "На распределении" переводит обращение в статус "Новое".

- После того, как обращение переведено в статус RESOLVED, обращение не доступно для редактирования (все поля read-only).

# ТИПЫ ОБРАЩЕНИЙ 

- Нужны для определения SLA обращения.

- Перечень типов: 
1. Консультация.
2. ЗНО - запрос на обслуживание - Проведение каких-либо работ Исполнителем (например, создание УЗ).
3. Инцидент - исправление ошибок, связанных с некорректной работой системы.

- Для каждого НО есть значение Тип-обращения по умолчанию, 
но Диспетчер и Исполнитель может менять это значение на любое из трех.

- Изменение типа влияет на SLA и отображается в карточке и журнале. Отдельное уведомление не отправляется.

---------------------------
# РАБОЧИЕ ГРУППЫ

- Исполнители поделены на рабочие группы, далее РГ. 

- Пример РГ - группа обслуживания корпоративного Help Desk-а "help.desk.service".

- У Диспетчеров так же есть своя группа - "help.desk.dispetcher".

- УЗ исполнителей и диспетчеров привязаны к РГ.

- Рабочие группы не привязаны к НО или РГ - при создании обращения система подставляет рекомендуемую РГ для каждого НО.
Диспетчер, при необходимости, сможет сменить РГ.

- Если в обращении указана "Рабочая группа", но не указан "Ответственный сотрудник",
обращение доступно для просмотра всем исполнителям, входящим в данную рабочую группу.

- Любой исполнитель из группы может взять обращение в работу и назначить себя ответственным.

- Рабочу группу может менять только Диспетчер, Исполнители не могут менять группу вручную.

- При смене РГ, если текущий assignee не входит в новую РГ, поле "Ответственный сотрудник" нужно очищать.

- Название группы отображается в обращении для Диспетчеров и Исполнителей, пользователь не видит названия группы.

# КОММЕНТАРИИ

- Для написания комментарии есть специальное поле, в которое можно ввести текст + кнопка "Добавить комментарий",
которая сохраняет этот текст в список комментариев данного обращения.

- На странице заявки список комментариев отображается в виде скрываемого/раскрываемого блока.

- Есть приватные комментарии - такие комментарии не видит инициатор.
Приватный комментарий может оставить Диспетчер или Исполнитель.
Чтобы сделать коммнтарий приватным - нужно установить галку под полем для написания комментария на странице обращения.

- Если обращение в статусе RESOLVED, комментарии добалвять нельзя.

# КНОПКИ В КАРТОЧКЕ ОБРАЩЕНИЯ

- "Подать обращение" - находится в самом низу страницы подачи обращения. Создает обращение.

- "Добавить комментарий" - находится под полем для ввода комментария. Сохраняет комментарий в список комментариев текущего обращения.

- "Отменить обращение" - Отображается у Инициатора в карточке зарегистрированного обращения.
Не меняет статус обращения, просто делает не приватную запись "Обращение отменено пользователем" в блоке комментариев.

- "Сохранить" - сохраняются изменения в отредактированных активных полях.
	
- "Вернуть на диспетчеров": 

1. Должен быть добавлен приватный комментарий в поле для ввода комментария, 
иначе появляется модальное окно с предупреждением ("Укажите причину возврата приватным комментарием"). 
1. В поле РГ подставляется группа "help.desk.dispetcher", обращение принимает статус "На распределении".
2. В БД меняются и статус, и группа.

- "Передать на группу" - используется для передачи обращения на РГ: если Диспетчер устанавливает ответственую группу в поле РГ,
после нажатия обращение попадет на группу исполнителей.

- "В работу" - если пользователь состоит в ответственной РГ данного обращения,
он автоматически подставляется в поле "Ответственный сотрудник", а обращение переходит в статус "В работе" (IN_PROGRESS).


- "Выполнить" - обращение принимает статус "Решено".

- "Кнопка открытия журнала" - открывает/закрывает окно журнала.

# ПОЛЯ НА ГЛАВНОЙ СТРАНИЦЕ

# ГЛАВНАЯ СТРАНИЦА ИНИЦИАТОРА

	Поля:
	1. Список обращений в виде таблицы:

	- Отображаются обращения данного инициатора.
	- Отфильтрованы по дате регистрации (по убыванию).
	- Инициатор может открыть карточку своих обращений.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.
	- Есть возможность прокрутки списка обращений на странице.

# ГЛАВНАЯ СТРАНИЦА ДИСПЕТЧЕРА

	Поля:
	1. Список обращений в виде таблицы:

	- Диспетчер работает только с обращениями, находящимися на группе диспетчеров.
	- Отображаются обращения на группе Диспетчеров со всеми статусами.
	- Отфильтрованы по дате регистрации (по убыванию).
	- Диспетчер может открыть карточку обращений, доступных его рабочей группе.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.

	- Диспетчеру доступна фильтрация по статусу обращения (NEW, IN_PROGRESS, UNASSIGNED, RESOLVED)

	- Есть возможность прокрутки списка обращений на странице.

# ГЛАВНАЯ СТРАНИЦА ИСПОЛНИТЕЛЯ

	Поля:
	1. Список обращений в виде таблицы:	
	- Исполнитель видит только обращения своей РГ.
	- По-умолчанию отображаются обращения по фильтру "Новые на моей РГ".

	- Исполнитель может отфильровать обращения: 
	1. Фильтр "Обращения на моей РГ" - отображать все обращения на его РГ (вне зависимости от отвественного) - статусы NEW, IN_PROGRESS, RESOLVED.  
	2. Фильтр "Новые на моей РГ" - отобажать только обращения в статусе NEW, поле "Ответственный сотрудник" не заполнено.
	3. Фильтр "Мои обращения" - отображать только те обращения, 
	где данный испольнитель указан как Ответственный сотрудник, статус обращения - NEW, IN_PROGRESS.

	- Отфильтрованы по дате регистрации (по убыванию).
	- Исполнитель может открыть карточку обращений своей рабочей группы.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.

	- Есть возможность прокрутки списка обращений на странице.
