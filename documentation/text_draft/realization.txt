# РАБОЧИЕ ГРУППЫ

- Исполнители поделены на рабочие группы, далее РГ. 

- РГ привязаны к БУ.

- Например - группа обслуживания корпоративного Help Desk-а "help.desk.service".

- У Диспетчеров так же есть своя группа - "help.desk.dispetcher".

- Ответственную группу в обращении назначает Диспетчер, Исполнители не могут менять группу вручную.

- Название группы отображается в обращении для Диспетчеров и Исполнителей, пользователь не видит названия группы.

- УЗ исполнителей привязаны к РГ.

---------------------------
# ОБЩАЯ СТРУКТУРА УСЛУГ

1. Первый уровень - "Раздел".

- Обозначает семейство вопросов, объединенных в один раздел.
Например: "Корпоративномый Help Desk".

2. Второй уровень - "Направление обслуживания" (далее НО).

- У кождого раздела есть перечень своих направлений обслуживания.
Пример направления обслуживания для раздела "Корпоративномый Help Desk": "Создать новый раздел".

4. Третий уровень - "Бизнес-услуга" (далее БО).

- Данная сущность нужна, чтобы диспетчер или исполнитель не менял направление обслуживания.
Вместо этого РГ привязаны к БУ.

- Изначально БУ подставляется в зависимости от НО, одна БУ может быть привязана к нескольким НО.

- Рабочие группы привязаны к БУ.

3. Четвертый уровень - "Тип обращения".

---------------------------
# РОЛИ

- Инициатор - Пользователь, который создает обращения. Видит историю своих обращений, может просматривать карточку обращения.

- Диспетчер - Проверяет поступающие обращения на корректность заполнения (требуется минимальная экспертиза), 
может передать обращение на ответственную РГ или закрыть обращение.

- Испольнитель - Проверяет поступающее обращение, может вернуть обращение диспетчерам,
может добавлять комментарии, может установить статус "Решено".

---------------------------
# СТАТУСЫ: 

- NEW - Новое(пользователь создал обращение и оно находится на диспетчере). 
- IN_PROGRESS - В работе (Выставляет исполнитель, когда берет обращение в работу).
- UNASSIGNED - На распределении (Исполнитель вернул на диспетчеров).
- RESOLVED - Решено. 

# Маршруты:
# Роль | Начальное состояние | Конечное состояние |

# ТИПЫ ОБРАЩЕНИЙ 

- Перечень типов: 
1. Консультация - SLA - 2 рабочих дня.
2. ЗНО - запрос на обслуживание - Проведение каких-либо работ Исполнителем (например, создание УЗ). SLA - 3 рабочих дня.
3. Инцидент - исправление ошибок, связанных с некорректной работой системы. SLA - 3 рабочих дня.

- Тип обращения привязан к БУ.

- Может быть изменен исполнителем или диспетчером вне зависимости от БУ.

# КНОПКИ В КАРТОЧКЕ ОБРАЩЕНИЯ

- "Подать обращение" - находится в самом низу страницы подачи обращения. Создает обращение.

- "Добавить комментарий" - находится под полем для ввода комментария. Сохраняет комментарий в список комментариев текущего обращения.

- "Отменить обращение" - Отображается у Инициатора в карточке зарегистрированного обращения.
Не меняет статус обращения, просто делает не приватную запись "Обращение отменено пользователем" в блоке комментариев.
	
- "Вернуть на диспетчеров" - в поле РГ подставляется группа "help.desk.dispetcher". 
Если при этом заполнен комментарий, обращение сохраняется, обращение попадает на группу Диспетчеров.
Если не заполнен комментарий, система подсвечивает это сообщением "Оставьте причину возврата приватным комментарием".
Обращение принимает статус "На распределении".

- "Сохранить" - сохраняются изменения в отредактированных активных полях.
Также данный механизм используется для передачи обращения на РГ: если Диспетчер или Исполнитель устанавливает ответственую группу в поле РГ,
после нажатия "Сохранить" обращение попадет на группу исполнителей (+ сохранятся все внесенные изменеения в обращении).

- "Выполнить" - обращение принимает статус "Решено".

- "Кнопка открытия журнала" - открывает/закрывает окно журнала.
