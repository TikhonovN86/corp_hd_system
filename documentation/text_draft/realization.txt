## РОЛИ ##

- Инициатор - Пользователь, который создает обращения. Видит историю своих обращений, может просматривать карточку обращения,
в которых он является инициатором. Обращения от других инициаторов не доступны для просмотра и редактирования.

- Диспетчер - Проверяет поступающие обращения на корректность заполнения (требуется минимальная экспертиза), 
может передать обращение на ответственную РГ или закрыть обращение, которое находится на "help.desk.dispetcher".

- Испольнитель - Проверяет поступающее обращение, может вернуть обращение диспетчерам,
может добавлять комментарии, может установить статус "Решено".

## ОБЩАЯ СТРУКТУРА УСЛУГ ##

1. Раздел.

- Обозначает семейство вопросов, объединенных в один раздел.
Например: "Корпоративномый Help Desk".

2. Направление обслуживания (далее НО).

- У кождого раздела есть перечень своих направлений обслуживания.
Пример направления обслуживания для раздела "Корпоративномый Help Desk": "Создать новый раздел".

3. Тип обращения.

- Подробнее в # ТИП ОБРАЩЕНИЙ.

4. Статус обращения.

- Подробнее в # СТАТУСЫ.

5. Рабочая группа (Далее РГ)

- Подробнее в # РАБОЧИЕ ГРУППЫ

---------------------------
## СТАТУСЫ ##

- Статус не отражает владельца процесса, за владельца отвечает поле "Рабочая группа".

- NEW - Новое  - пользователь создал обращение, оно не принято в рабоу. Изначально попадает на группу диспетчеров.
- IN_PROGRESS - В работе.
- UNASSIGNED - На распределении (Исполнитель вернул на диспетчеров). Допустим только при work_group = help.desk.dispetcher.
- RESOLVED - Решено.

Инициатор:
Новое

# ВАЖНЫЕ МОМЕНТЫ:

- Диспетчер из статуса "На распределении" переводит обращение в статус "Новое", это происходит при нажатии кнопки "Передать обращение".

- После того, как обращение переведено в статус RESOLVED, обращение не доступно для редактирования (все поля read-only).

## ТИПЫ ОБРАЩЕНИЙ ##

- Нужны для определения SLA обращения.

- Перечень типов: 

	1. Консультация.
	2. ЗНО - запрос на обслуживание - Проведение каких-либо работ Исполнителем (например, создание УЗ).
	3. Инцидент - исправление ошибок, связанных с некорректной работой системы.

- Для каждого НО есть значение Тип-обращения по умолчанию, 
но Диспетчер и Исполнитель может менять это значение на любое из трех.

- Изменение типа влияет на SLA и отображается в карточке и журнале. Отдельное уведомление не отправляется.

## РАБОЧИЕ ГРУППЫ ##

- Исполнители поделены на рабочие группы, далее РГ. 

- Пример РГ - группа обслуживания корпоративного Help Desk-а "help.desk.service".

- У Диспетчеров так же есть своя группа - "help.desk.dispetcher".

- УЗ исполнителей и диспетчеров привязаны к РГ.

- Рабочие группы не привязаны к НО или РГ - при создании обращения система подставляет рекомендуемую РГ для каждого НО.
Диспетчер, при необходимости, сможет сменить РГ.

- Если у обращения указан "Ответственный сотрудник", 
остальне участники данной РГ могут просматривать данное обращение 
(При изменении полей происходит сверка с действующим ответственным).

- Любой исполнитель из группы может взять обращение в работу и назначить себя ответственным.

- Рабочу группу может менять только Диспетчер, Исполнители не могут менять группу вручную.

- При смене РГ, если текущий assignee не входит в новую РГ, поле "Ответственный сотрудник" нужно очищать.

- Название группы отображается в обращении для Диспетчеров и Исполнителей, пользователь не видит названия группы.

- Каждый участник рабочей группы может указать в поле "Ответственный сотрудник" любого участника данной РГ, не только себя.

## КОММЕНТАРИИ ##

- Для написания комментарии есть специальное поле, в которое можно ввести текст + кнопка "Добавить комментарий",
которая сохраняет этот текст в список комментариев данного обращения.

- На странице заявки список комментариев отображается в виде скрываемого/раскрываемого блока.

- Есть приватные комментарии - такие комментарии не видит инициатор.
Приватный комментарий может оставить Диспетчер или Исполнитель.
Чтобы сделать коммнтарий приватным - нужно установить галку под полем для написания комментария на странице обращения.

- Если обращение в статусе RESOLVED, комментарии добалвять нельзя.

## КНОПКИ В КАРТОЧКЕ ОБРАЩЕНИЯ ##

- "Подать обращение" - находится в самом низу страницы подачи обращения. Создает обращение.

- "Добавить комментарий" - находится под полем для ввода комментария. Сохраняет комментарий в список комментариев текущего обращения.

- "Отменить обращение" - Отображается у Инициатора в карточке зарегистрированного обращения.
Не меняет статус обращения, просто делает не приватную запись "Обращение отменено пользователем" в блоке комментариев.

- "Сохранить":

	Сценарий №1 - Используется Диспетчером или Исполнителем при изменении поля "Тип обращения".
	
		Условия успешного выполнения:
		1. Диспетчер или исполнитель выбрал новое значение из выпадающего списка в поле "Тип обращения" и нажал кнопку "Сохранить". 

		Поведение системы:
		1. У обращения меняется тип, пересчитывается SLA, делается запись в жрурнал.

	Сценарий №2 - Используется Диспетчером или Исполнителем при выборе ответственного сотрудника в поле "Ответственный сотрудник".	

		Условия успешного выполнения:
		1. Диспетчер или исполнитель выбрал новое значение из выпадающего списка в поле "Ответственный сотрудник" и нажал кнопку "Сохранить". 

		Поведение системы:
		1. У обращения меняется ответственный сотрудник, делается запись в жрурнал.
		
	Альтернативный сценарий:	
		
		1. Если значения в полях "Тип обращения" или "Ответственный сотрудник" не изменились, кнопка ничего не делает.
	
- "Вернуть на диспетчеров": 

	Используется Исполнителем для возврата обращения на группу диспетчером.

	Условия успешного выполнения:
	1. Должен быть добавлен приватный комментарий в поле для ввода комментария. 
	Если сообщения нет или оно не приватное - появляется модальное окно с предупреждением ("Укажите причину возврата приватным комментарием"). 

	Поведение системы:
	1. В поле РГ подставляется группа "help.desk.dispetcher", поле "Ответственный сотрудник" очищается, 
	обращение принимает статус "На распределении".
	2. В БД меняюется статус (на UNASSIGNED), группа (на help.desk.dispetcher), атрибут ответственный сотрудник очищается // Пока условно.

- "Передать обращение":

	Используется для передачи обращения диспетчером на ответственную РГ. 
	
	Условия успешного выполнения:
	1. В поле "Рабочая группа" должна быть установлена ответственная группа, отличная от "help.desk.dispetcher".
	Если в поле "Рабочая группа" отсутствует РГ или указана "help.desk.dispetcher" - 
	появляется модальное окно с надписью "Для передачи обращения укажите ответсвернную рабочую группу".

	Поведение системы:
	1. Если обращение имеет статус NEW - после нажатия кнопки "Передать обращение", поле "Ответственный сотрудник" очищается, 
	обращение попадет на группу исполнителей в статусе NEW.
	2. Если обращение имеет статус IN_PROGRESS - после нажатия кнопки "Передать обращение", поле "Ответственный сотрудник" очищается, 
	обращение попадет на группу исполнителей в статусе NEW.
	2. Если обращение имеет статус UNASSIGNED - после нажатия кнопки "Передать обращение", поле "Ответственный сотрудник" очищается, 
	обращение попадет на группу исполнителей в статусе NEW.

- "В работу": 
	
	Используется Диспетчером или Исполнителем для перевода обращения в статус IN_PROGRESS.
	
	Условия успешного выполнения:
	
	1. Если поле "Ответственный сотрудник" пустое и Исполнитель состоит в ответственной РГ данного обращения, а так же обращение в статусе NEW, 
	Исполнитель автоматически подставляется в поле "Ответственный сотрудник", а обращение переходит в статус "В работе" (IN_PROGRESS).
	2. Если поле "Ответственый сотрудник" заполнено, и значение совпадает с Исполнителем, который нажимает на кнопку, 
	а так же обращение в статусе NEW, обращение переходит в статус "В работе" (IN_PROGRESS).

	3. Если обращение уже находится в статусе "IN_PROGRESS" - ничего не происходит.

	4. Если поле "Ответственный сотрудник" пустое и обращение находится на группе диспетчеров 
	(в поле "Рабочая группа" указана РГ help.desk.dispetcher), а так же обращение находится в статусе NEW или UNASSIGNED, 
	диспетчер автоматически подставляется в поле "Ответственный сотрудник", 
	а обращение переходит в статус "В работе" (IN_PROGRESS).
	5. Если поле "Ответственный сотрудник" заполнено значением, которое совпадает с диспетчером, нажимающим кнопку "В работе"
	и обращение находится на группе диспетчеров (в поле "Рабочая группа" указана РГ help.desk.dispetcher), 
	а так же обращение находится в статусе NEW или UNASSIGNED, обращение переходит в статус "В работе" (IN_PROGRESS).

- "Выполнить" - обращение принимает статус "Решено".

	Условия успешного выполнения:
	1. Обращение находится в статусе "IN_PROGRESS".
	Если обращение наодится в статусе NEW или UNASSIGNED, возникает модальное окно "Необходимо взять обращение в работу".
	2. В поле для комментариев должен быть записан не приватный комментарий. 
	Кнопка "Выполнить" автоматически сохраняет этот текст как обычный комментарий, и только потом закрывает обращение. 
	Если поле для комментария пустое или указан приватный комментарий - обращение не закрывается, 
	появляется модальное окно "Для решения обращения добавьте не приватный комментарий".
	3. Должно быть заполнено поле "Ответственный сотрудник" - нужно указать исполнителя из ответственной РГ, 
	в т.ч. диспетчера, если РГ - "help.desk.dispetcher".
	Иначе обращение не закрывается, появляется модальное окно "Для решения обращения укажите ответственного сотрудника".

- "Кнопка открытия журнала" - открывает/закрывает окно журнала.

## ПОЛЯ НА ГЛАВНОЙ СТРАНИЦЕ ##

# ГЛАВНАЯ СТРАНИЦА ИНИЦИАТОРА

	Поля:
	1. Список обращений в виде таблицы:

	- Отображаются обращения данного инициатора.
	- Отфильтрованы по дате регистрации (по убыванию).
	- Инициатор может открыть карточку своих обращений.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.
	- Есть возможность прокрутки списка обращений на странице.

# ГЛАВНАЯ СТРАНИЦА ДИСПЕТЧЕРА

	Поля:
	1. Список обращений в виде таблицы:

	- Диспетчер на главной странице видит только обращения РГ диспетчеров.
	- Отображаются обращения на группе Диспетчеров со всеми статусами.
	- Отфильтрованы по дате регистрации (по убыванию).
	- Диспетчер может открыть только карточку обращения, которое находится на его РГ.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.
	- Есть возможность прокрутки списка обращений на странице.

	- Диспетчеру доступна фильтрация по статусу обращения (NEW, IN_PROGRESS, UNASSIGNED, RESOLVED).

# ГЛАВНАЯ СТРАНИЦА ИСПОЛНИТЕЛЯ

	Поля:
	1. Список обращений в виде таблицы:	
	- Исполнитель видит только обращения своей РГ.
	- По-умолчанию отображаются обращения по фильтру "Новые на моей РГ".
	- Отфильтрованы по дате регистрации (по убыванию).
	- Исполнитель может открыть карточку только тех обращений, которые находятся в ответственности его РГ.
	- Отображаются столбцы: номер обращения, статус обращения, дата регистрации.
	- Есть возможность прокрутки списка обращений на странице.

	- Исполнитель может отфильровать обращения: 
	1. Фильтр "Обращения на моей РГ" - отображать все обращения на его РГ (вне зависимости от отвественного) - статусы NEW, IN_PROGRESS, RESOLVED.  
	2. Фильтр "Новые на моей РГ" - отобажать только обращения в статусе NEW, поле "Ответственный сотрудник" не заполнено.
	3. Фильтр "Мои обращения" - отображать только те обращения, 
	где данный испольнитель указан как Ответственный сотрудник, статус обращения - NEW, IN_PROGRESS.
